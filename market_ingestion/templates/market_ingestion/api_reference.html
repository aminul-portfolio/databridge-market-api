Overview

Read-only operational API

These endpoints expose data already stored in the DataBridge platform database. They do not trigger live provider fetches and do not allow writes.

This API is intended for operational inspection, portfolio review, and lightweight integration testing. List endpoints are paginated, detail endpoints return a single record, and example detail IDs shown below are illustrative only.

Query Parameters

Supported Filters

Use query parameters to narrow list endpoints. Page size is capped in the backend.

  • source — provider name, such as yfinance
  • symbol — instrument symbol, such as BTC-USD
  • timeframe — interval value, such as 1h
  • status — available on ingestion runs, such as success
  • source_file — available on journal entries, such as trading-journal.csv
  • page — page number for paginated results
  • page_size — requested page size, capped in the backend
Runs: status=success Runs: source=yfinance Snapshots: symbol=BTC-USD Bars: yfinance + BTC-USD + 1h Journal: symbol=BTC-USD Journal: source_file
